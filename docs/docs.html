RecordIt Documentation

Professional screen recording and whiteboard for Windows and beyond

RecordIt is a full-featured screen recorder and collaborative whiteboard application. It ships as two apps with an identical Alvonia UI design system:

  • RecordIt WinUI — Native Windows app built with WinUI 3 and Windows App SDK
  • RecordIt Alvonia — Cross-platform Electron + React app for Windows, macOS, and Linux

Quick Start

The fastest way to get started:

# Clone the repository git clone https://github.com/dotnetappdev/recordit.git cd recordit # Install Electron app dependencies cd alvonia-ui npm install # Start in development mode npm run electron:dev

Or for the WinUI 3 app, open winui/RecordIt.sln in Visual Studio 2022 and build the solution.

Architecture Overview

The repository contains three app implementations sharing one core library and design system:

recordit/ ├── winui/ # WinUI 3 / .NET 10 (Windows-native) │ └── RecordIt/ │ ├── Pages/ # RecordPage, WhiteboardPage, LibraryPage, SettingsPage │ ├── Services/ │ │ └── AudioMeterService.cs # Core Audio COM interop │ └── Styles/AlvoniaTheme.xaml # XAML design tokens + styles ├── RecordIt.Avalonia/ # Avalonia UI / .NET (cross-platform) │ └── ... # Mirrors WinUI pages and styles ├── RecordIt.Core/ # Shared .NET core library │ ├── Services/ │ │ └── ScreenRecordingService.cs # ffmpeg backend │ └── Models/ ├── alvonia-ui/ # Electron + React (cross-platform) │ ├── src/components/ # Page components │ └── src/styles/global.css # Alvonia UI CSS tokens └── docs/ # GitHub Pages static site

Studio Interface

The recording page uses a 5-panel dockable layout inspired by professional broadcast software, with RecordIt's own Alvonia UI visual identity:

  • Scenes panel — add/remove/reorder named scenes; quick-switch bar overlaid on the preview
  • Sources panel — add capture sources (screen, window, camera, audio input, desktop audio) with icon-per-type list
  • Audio Mixer panel — live VU meters + volume faders + mute per channel; + button to add any detected audio device
  • Options panel — resolution, frame rate, camera overlay device, and mic device selectors
  • Controls panel — Start/Stop Recording, Start/Stop Preview, Settings, Library buttons

Panel visibility is toggled from View → Scenes / Sources / Audio Mixer / Transitions / Controls. All panels are present in both WinUI 3 and Avalonia versions.

Audio Mixer

The Audio Mixer provides real-time VU metering and per-channel volume control using Windows Core Audio:

// AudioMeterService uses IAudioMeterInformation COM interface // Peak values polled at 50 ms via DispatcherTimer var dPeak = _audioMeter.GetDesktopPeak(); // IMMDeviceEnumerator → eRender endpoint var mPeak = _audioMeter.GetMicPeak(); // IMMDeviceEnumerator → eCapture endpoint

Each channel strip contains:

  • Channel name label with a colour-coded type dot (cyan = desktop, indigo = mic, violet = other)
  • Dual VU bars (L + R) with green → yellow → red gradient, height updated from peak value × container height
  • dB readout label (0.0 dB at full volume, −∞ dB at zero)
  • Horizontal volume slider (0.0 – 1.0, 0.01 step), value applied as -af volume=X in ffmpeg
  • Mute toggle button — zeros both VU bars when active

Additional channels are added by clicking + in the Audio Mixer toolbar, which shows a dialog listing all detected dshow audio devices.

Screen Recording

The WinUI 3 and Avalonia apps use ffmpeg via ScreenRecordingService in RecordIt.Core:

  • Video capture: -f gdigrab -i desktop (primary display) or all screens
  • Desktop audio: -f wasapi -loopback for system audio without virtual audio cables
  • Microphone: -f dshow -i audio=<device> with correct stream index tracking
  • Webcam overlay: -f dshow -i video=<device> scaled to 320×180 and composited bottom-right
  • Audio stream index fix: input index is now tracked dynamically — was previously hardcoded to 2 which caused silent recordings when no webcam was selected

Device enumeration uses ProbeDevicesAsync() which parses ffmpeg's -list_devices output and returns separate VideoDevices and AudioDevices lists — previously all device names were mixed together.

Electron: Uses Chromium's desktopCapturer API with the Web MediaRecorder API for cross-platform capture.

Whiteboard

The whiteboard provides an MS Teams-style collaborative canvas experience:

  • Pen, highlighter, eraser with configurable brush sizes
  • Shape tools: line, rectangle, ellipse, triangle
  • Text and sticky note tools
  • 10-color palette + custom color picker
  • Participants panel showing active collaborators
  • Zoom in/out controls (25%–400%)
  • Export canvas to PNG

Electron implementation: Uses HTML5 Canvas with React for rendering. All drawing operations are immediate with no framework overhead.

WinUI 3 implementation: Uses WinUI Canvas with XAML shapes for vector-quality rendering.

Alvonia UI Design System

Alvonia UI is the shared design language powering both apps. Key design tokens:

/* Color tokens */ --color-brand-primary: #6366f1 /* Indigo */ --color-brand-secondary: #8b5cf6 /* Violet */ --color-brand-accent: #06b6d4 /* Cyan */ --color-bg-base: #0f0f0f --color-bg-surface: #1a1a1a --color-text-primary: #f5f5f5 --color-recording: #ef4444 /* Spacing scale: 4, 8, 12, 16, 20, 24, 32... */ /* Border radius: 4, 8, 12, 16, 9999 (full) */ /* Typography: Inter, 11–28px scale */

In XAML, these are defined in Styles/AlvoniaTheme.xaml as Color, SolidColorBrush, and Style resources.

MSIX Packaging

The RecordIt.Package project handles MSIX creation:

# Build MSIX from command line cd winui msbuild RecordIt.Package\RecordIt.Package.wapproj ^ /p:Configuration=Release /p:Platform=x64 ^ /p:UapAppxPackageBuildMode=StoreUpload

For the Electron app, MSIX/AppX is generated by electron-builder:

cd alvonia-ui npm run electron:build:win # Builds NSIS + AppX
