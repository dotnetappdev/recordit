RecordIt Documentation

Professional screen recording and whiteboard for Windows and beyond

RecordIt is a full-featured screen recorder and collaborative whiteboard application. It ships as two apps with an identical Alvonia UI design system:

  • RecordIt WinUI — Native Windows app built with WinUI 3 and Windows App SDK
  • RecordIt Alvonia — Cross-platform Electron + React app for Windows, macOS, and Linux

Quick Start

The fastest way to get started:

# Clone the repository git clone https://github.com/dotnetappdev/recordit.git cd recordit # Install Electron app dependencies cd alvonia-ui npm install # Start in development mode npm run electron:dev

Or for the WinUI 3 app, open winui/RecordIt.sln in Visual Studio 2022 and build the solution.

Architecture Overview

The repository is organized into three main areas:

recordit/ ├── alvonia-ui/ # Electron + React app (cross-platform) │ ├── electron/ # Main process + preload scripts │ ├── src/ # React UI (TypeScript) │ │ ├── components/ # Page and UI components │ │ └── styles/ # Alvonia UI CSS design system │ └── package.json ├── winui/ # WinUI 3 C# app (Windows-native) │ ├── RecordIt/ # Main WinUI 3 project │ │ ├── Pages/ # XAML pages (Record, Whiteboard, Library, Settings) │ │ ├── Services/ # Screen recording service │ │ └── Styles/ # AlvoniaTheme.xaml (XAML design system) │ └── RecordIt.Package/ # MSIX packaging project └── docs/ # GitHub Pages static documentation site

Screen Recording

RecordIt uses platform-appropriate APIs for screen capture:

  • WinUI 3: Windows Graphics Capture API (GraphicsCaptureItem), Media Foundation encoding
  • Electron: Chromium's desktopCapturer API, Web MediaRecorder API

Both support selecting screens, windows, or regions, recording audio from the microphone, and exporting to MP4 or WebM.

Whiteboard

The whiteboard provides an MS Teams-style collaborative canvas experience:

  • Pen, highlighter, eraser with configurable brush sizes
  • Shape tools: line, rectangle, ellipse, triangle
  • Text and sticky note tools
  • 10-color palette + custom color picker
  • Participants panel showing active collaborators
  • Zoom in/out controls (25%–400%)
  • Export canvas to PNG

Electron implementation: Uses HTML5 Canvas with React for rendering. All drawing operations are immediate with no framework overhead.

WinUI 3 implementation: Uses WinUI Canvas with XAML shapes for vector-quality rendering.

Alvonia UI Design System

Alvonia UI is the shared design language powering both apps. Key design tokens:

/* Color tokens */ --color-brand-primary: #6366f1 /* Indigo */ --color-brand-secondary: #8b5cf6 /* Violet */ --color-brand-accent: #06b6d4 /* Cyan */ --color-bg-base: #0f0f0f --color-bg-surface: #1a1a1a --color-text-primary: #f5f5f5 --color-recording: #ef4444 /* Spacing scale: 4, 8, 12, 16, 20, 24, 32... */ /* Border radius: 4, 8, 12, 16, 9999 (full) */ /* Typography: Inter, 11–28px scale */

In XAML, these are defined in Styles/AlvoniaTheme.xaml as Color, SolidColorBrush, and Style resources.

MSIX Packaging

The RecordIt.Package project handles MSIX creation:

# Build MSIX from command line cd winui msbuild RecordIt.Package\RecordIt.Package.wapproj ^ /p:Configuration=Release /p:Platform=x64 ^ /p:UapAppxPackageBuildMode=StoreUpload

For the Electron app, MSIX/AppX is generated by electron-builder:

cd alvonia-ui npm run electron:build:win # Builds NSIS + AppX

Contributing

Contributions are welcome! Please follow these steps:

# Fork the repo, then: git clone https://github.com/YOUR_USERNAME/recordit.git cd recordit git checkout -b feature/my-feature # Make your changes, commit, and open a PR git commit -m "feat: add amazing feature" git push origin feature/my-feature

Please ensure your changes maintain UI parity between the WinUI 3 and Electron apps, and follow the Alvonia UI design system tokens.
